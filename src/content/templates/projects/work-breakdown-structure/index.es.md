Para organizar un proyecto inicialmente abstracto en elementos planificables, puede utilizar una estructura de desglose del trabajo para dividir el proceso en subtareas y paquetes de trabajo. Antes de que su equipo elabore paquetes de trabajo individuales, debe definir las subtareas en las que se divide el proyecto. Puede organizar estas agrupaciones de nivel superior según fases, funciones u objetos.

Una vez que el objetivo y las subtareas de tu proyecto están claros, es hora de definir los paquetes de trabajo. Describe la tarea, añade una persona responsable, una ubicación, una hora de inicio y un plazo. También puedes anotar las subtareas y las horas de trabajo. Como las fases del proyecto (Tabla 2) están vinculadas a los paquetes de trabajo (Tabla 1), puedes asignar cada paquete de trabajo a una fase y obtener así una visión general del progreso del proyecto y de las horas trabajadas.

Utilice distintas vistas que filtren las tareas de cada responsable para mostrar listas de tareas pendientes personalizadas para cada miembro del equipo. Aprovecha el plugin de calendario y cronograma para asignar periodos de tiempo y controlar los plazos.
