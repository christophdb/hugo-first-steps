Чтобы использовать общую запись в другой базе, вы можете импортировать соответствующее табличное представление общей записи в нужную базу. Для этого создайте новую таблицу в базе с соответствующим набором данных. Затем вы можете работать с данными общего набора данных или табличного представления, включая определенные условия фильтрации и сортировки, в выбранной базе.

## Требования

Чтобы иметь возможность создать новую таблицу в базе с общей записью данных, база должна удовлетворять одному из следующих двух требований:

1. База должна принадлежать к **группе**, в которой была **создана** запись общих данных.
2. База должна принадлежать к **группе**, для которой заранее была **выпущена** запись общих данных.

{{< warning  headline="Важное замечание"  text="База **должна** удовлетворять одному из этих двух требований, чтобы в ней можно было создать новую таблицу с общей записью данных. Однако **если** к базе не относится ни одно из этих двух условий, создать новую таблицу с общей записью данных **невозможно**." />}}

## Создайте новую таблицу с общей записью

![Создайте новую таблицу с общей записью](https://seatable.io/wp-content/uploads/2022/11/image-how-to-create-a-new-table-with-a-common-dataset.png)

1. Откройте **стол**, находящийся в базе, которая соответствует одному из описанных выше **требований**.
2. Нажмите на **символ \[+\]**, чтобы создать новую таблицу.
3. Нажмите **Импорт общей записи**.
4. Выберите **общую запись**.
5. Затем общий набор данных импортируется в вашу базу в виде **новой таблицы**.

Если **импорт** общего набора данных прошел **успешно**, появится следующее сообщение:

![При успешном импорте общего набора данных появляется следующее сообщение](images/Erfolgsmeldung-beim-Import-eines-GD-.png)

## Характеристики таблиц, созданных с общим набором данных

Таблицы, созданные с помощью записи общих данных, можно отличить от других таблиц по **значку стека** перед именем таблицы. Если навести указатель мыши на значок, то короткий **информационный текст** также укажет, что таблица была создана с помощью записи общих данных.

![Значок перед названием таблиц общего набора данных.](https://seatable.io/wp-content/uploads/2022/11/icon-table-common-dataset-1.png)

Еще одна особенность - **символ** на значке столбцов, содержащих данные из **общего** набора данных. Столбцы таблицы, **не** содержащие данных из общей записи данных, поэтому **не** имеют такой маркировки на значке столбца.

![Столбец обычного набора данных (слева) и обычный столбец (справа)](https://seatable.io/wp-content/uploads/2022/11/icons-of-the-column-of-a-common-dataset-2.png)

## Последствия для обработки данных

Главное правило, которого следует придерживаться при работе с общими записями, заключается в том, что информация течет только в **одном направлении**. Конкретно это означает, что изменения в общих записях и в таблицах, созданных с помощью общей записи, имеют эффект только в одном направлении - а именно в сторону **таблиц**.

Если вы вносите изменения в табличное представление, которое вы опубликовали как **общий набор данных**, эти изменения также принимаются во всех таблицах, которые были созданы с помощью соответствующего общего набора данных. Необходимым условием для этого является **синхронизация соответствующих таблиц с набором данных**, которая [может быть выполнена автоматически или вручную](https://seatable.io/ru/docs/gemeinsame-datensaetze/synchronisation-eines-gemeinsamen-datensatzes/).

С другой стороны, если вы вносите изменения в данные из общего набора данных в таблице, которую вы создали с помощью общего набора данных, эти изменения **не** влияют на структуру общего набора данных. Затем изменения применяются **исключительно** к вашей таблице и будут **перезаписаны** при следующей **синхронизации** с набором данных.

Однако во время синхронизации в таблице остаются только **измененные имена столбцов**. Поэтому вы можете переименовать столбцы записи общих данных в своей таблице по своему усмотрению.
