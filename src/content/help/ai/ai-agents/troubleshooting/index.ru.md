Здесь вы найдёте решения самых распространённых проблем при использовании ИИ-агентов с MCP-сервером SeaTable. Эти рекомендации применимы ко всем поддерживаемым клиентам — Claude Desktop, Claude Code и Cursor.

## Проблемы с подключением

### «Connection timeout» или сервер недоступен

MCP-сервер по адресу `mcp.seatable.com` должен быть доступен через Интернет.

- Проверьте подключение к Интернету.
- Откройте `https://mcp.seatable.com/health` в браузере. При работающем подключении вы получите ответ `{"status":"ok","version":"1.0.2"}`.
- Если вы находитесь за прокси-сервером или межсетевым экраном, убедитесь, что исходящие HTTPS-соединения к `mcp.seatable.com` разрешены.

### Подключение обрывается через некоторое время

MCP-подключения имеют ограниченную длительность сессии. Если агент внезапно теряет доступ через некоторое время, сессия истекла.

- **Claude Desktop**: Перезапустите Claude Desktop или проверьте статус подключения в настройках.
- **Claude Code**: Введите `/mcp` и выберите **Reconnect**. Claude Code в настоящее время не переподключается автоматически при истечении сессии.
- **Cursor**: Откройте **Settings** → **Tools & MCP** и проверьте статус. Если отображается красная точка, нажмите на запись сервера, чтобы восстановить подключение.

## Проблемы с токеном

### «Invalid API token»

API-токен не принимается.

- Проверьте, что токен скопирован полностью — без пробелов в начале или конце.
- Убедитесь, что вы используете **API-токен** (токен базы), а не токен аккаунта.
- Проверьте в SeaTable, существует ли токен. Удалённый токен сразу становится недействительным.

### «Permission denied» или нет прав на запись

Агент может читать данные, но не может их изменять.

- Вероятно, вы используете **токен только для чтения**. Создайте новый токен с правами на чтение и запись, если хотите предоставить агенту доступ на запись.
- Замените токен в конфигурации клиента и перезапустите подключение.

## Проблемы, специфичные для клиентов

### Claude Desktop

| Проблема | Решение |
|---|---|
| Значок инструментов не отображается | Проверьте, что файл `claude_desktop_config.json` содержит валидный JSON, и перезапустите Claude Desktop. |
| Сервер не загружается | Убедитесь, что Node.js установлен. Конфигурация требует `npx` для запуска `mcp-remote`. |
| Несколько баз не работают | Каждая запись в `mcpServers` должна иметь уникальное имя (например, `seatable-crm`, `seatable-projects`). |

### Claude Code

| Проблема | Решение |
|---|---|
| MCP-сервер не распознаётся | Проверьте с помощью `claude mcp list`, зарегистрирован ли сервер. Если нет, выполните команду `claude mcp add` повторно. |
| Подключение потеряно после неактивности | Введите `/mcp` и выберите **Reconnect**. Это известное поведение — Claude Code не переподключается автоматически. |
| Неправильный URL сервера | Удалите сервер командой `claude mcp remove seatable` и добавьте его снова с правильным URL. |

### Cursor

| Проблема | Решение |
|---|---|
| Красная точка рядом с сервером | URL или токен указаны неверно. Проверьте оба параметра в **Settings** → **Tools & MCP**. |
| Cursor не использует MCP-сервер | Убедитесь, что вы используете **чат** (а не встроенное редактирование). MCP-сервер используется только в режиме чата. |
| Сервер не отображается в списке | Проверьте, что файл `.cursor/mcp.json` содержит валидный JSON и что запись сервера правильно отформатирована. |

## Общие советы

**Проверьте конфигурацию.** Самая частая причина ошибок — опечатки в конфигурации: пропущенная кавычка, лишняя запятая или неправильно скопированный токен. Проверьте конфигурацию внимательно.

**Перезапуск часто помогает.** Многие проблемы с подключением решаются перезапуском клиента.

**Создайте токен заново.** Если вы не уверены в правильности токена, просто создайте новый токен в SeaTable. Это быстрее, чем искать ошибку.

**Один MCP-сервер за раз.** Если вы подключаете несколько баз и возникают проблемы, сначала протестируйте с одной базой, прежде чем добавлять другие.
