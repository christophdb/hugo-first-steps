С помощью функции обработки данных можно выполнять различные **операции**. Вы можете выполнять математические вычисления по всем значениям в столбце, а также соотносить значения, когда записи в двух разных таблицах идентичны.

Вы выполняете операции по обработке данных либо **вручную**, либо с **помощью автоматизации**. В этой статье объясняется, как именно происходит выполнение вручную. Подробную информацию о выполнении с помощью автоматизации можно найти [здесь]({{< relref "help/base-editor/automationen/beispiel-datenverarbeitungsoperation-per-automation-ausfuehren" >}}).

## Выполнение операции по обработке данных

![Выполнение действия по обработке данных](images/create-an-data-processing-action-1.jpg)

1. Откройте любую **таблицу**, в которой вы хотите выполнить операцию обработки данных.
2. Нажмите на **три точки** в опциях просмотра.
3. Нажмите на **Обработка данных**.
4. Наведите курсор мыши на уже созданную **операцию обработки данных**, которую вы хотите выполнить.
5. Нажмите на **значок воспроизведения**.

![Выполнение действия по обработке данных](images/run-data-processing-actions.jpg)

7. После успешного выполнения вместо игрового символа появляется **зеленая галочка**.

![Зеленая галочка для подтверждения того, что действие по обработке данных было выполнено.](images/conformation-for-the-runof-a-data-processing-action.jpg)

{{< warning  headline="Важное замечание" >}}

Операции по обработке данных можно выполнять не только **вручную**, но и с [помощью автоматизации]({{< relref "help/base-editor/automationen/beispiel-datenverarbeitungsoperation-per-automation-ausfuehren" >}}). Если вам интересно, прочитайте об этом в статье по ссылке.

{{< /warning >}}

## Последствия казни

Каждый раз, когда выполняется операция обработки данных, **результаты** записываются в **столбец результатов** независимо от этого. Если вы не хотите перезаписывать данные, следует заранее [создать новый пустой столбец]({{< relref "help/base-editor/spalten/hinzufuegen-einer-spalte" >}}) для результатов.

В отличие от столбцов **формул**, которые постоянно отслеживают соответствующие столбцы, **столбцы результатов** не обновляются. Изменения значений в столбце источника не влияют на значения в столбце результата без [автоматизации]({{< relref "help/base-editor/automationen/beispiel-datenverarbeitungsoperation-per-automation-ausfuehren" >}}) или повторного выполнения.

Поэтому во избежание недоразумений мы рекомендуем **не** вносить никаких изменений вручную в соответствующие столбцы и в целях безопасности [заблокировать]({{< relref "help/base-editor/spalten/spaltenberechtigungen-festlegen" >}}) после выполнения (требуется подписка Plus или Enterprise).

![Столбцы результатов действий по обработке данных, заблокированных для обработки ](images/locked-score-columns.jpg)

Помните, что вычисленные значения являются моментальным снимком на момент выполнения операции. Если вы не выполните операцию повторно, результаты могут быть устаревшими, если за это время значения в исходном столбце изменились.
