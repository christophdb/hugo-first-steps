{{< required-version "Предприятие" >}}

Вы можете **удалить** уже созданные **правила автоматизации** в любое время. Обратите внимание на то, как удаление влияет на различные **автоматические действия**.

## Чтобы удалить автоматизацию

![Удалить автоматизацию](images/delete-automation-rule.gif)

1. Откройте **Базу**, в которой Вы хотите удалить уже созданную автоматизацию.
2. В шапке базы нажмите на {{< seatable-icon icon="dtable-icon-rule">}}, а затем на **Правила автоматизации**.
3. Наведите указатель мыши на соответствующее правило автоматизации, нажмите на три точки, а затем на {{< seatable-icon icon="dtable-icon-delete" >}} **Удалить правило**.
4. Подтвердите нажатием **Удалить**.

{{< warning type="warning" headline="Deletion is final" text="После этого удаленное правило автоматизации **окончательно исчезнет и не может быть восстановлено**." />}}

## Эффекты

В зависимости от того, какие действия Вы определили для автоматизации, удаление правила автоматизации может иметь **различные последствия**. Однако, в принципе, после удаления определенное триггерное событие **не будет больше вызывать никаких действий** - если только оно не используется в качестве триггера для другой существующей автоматизации.

Особенно в случае сложных действий, таких как отправка электронной почты, скрипты Python или функции искусственного интеллекта, Вам следует рассмотреть возможность только [приостановить автоматизацию]({{< relref "help/base-editor/automationen/automationen-stoppen" >}}) вместо того, чтобы удалять ее навсегда.
