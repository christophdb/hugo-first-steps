[Markdown](https://en.wikipedia.org/wiki/Markdown) - это сильно упрощенный язык разметки, используемый для **форматирования текста**. SeaTable использует Markdown в различных местах для форматирования длинных текстов в визуально привлекательном виде.

Вам не обязательно быть знакомым с синтаксисом Markdown. Но независимо от того, используете ли вы Markdown впервые или уже имеете опыт работы с ним, эта статья поможет вам лучше понять, где и как использовать версию Markdown для SeaTable.

## SeaTable Markdown

В настоящее время существует пять областей, где вы можете использовать синтаксис Markdown в SeaTable. К ним относятся:

- Колонки типа [форматированного текста]({{< relref "help/base-editor/text-spalten/die-spalten-text-und-formatierter-text" >}})
- [Базовые описания]({{< relref "help/base-editor/weitere-optionen/wie-man-einer-base-eine-beschreibung-hinzufuegt" >}})
- Тексты электронной почты при отправке через [автоматизацию]({{< relref "help/base-editor/schaltflaechen/eine-e-mail-per-schaltflaeche-verschicken" >}})
- [Веб-формы]({{< relref "help/base-editor/webformulare/webformulare" >}})
- [Универсальные приложения]({{< relref "help/app-builder/app-verwaltung/universelle-app" >}})

Как только Markdown будет использован в SeaTable, вам будет предложена следующая маска ввода. Конечно, вы можете пометить слова или текстовые блоки как обычно, а затем выбрать нужное форматирование.

![Разметка в SeaTable](images/markdown-seatable.png)

Кроме того, форматирование Markdown можно вызвать непосредственно вводом определенных специальных символов. Просто введите специальный символ в поле ввода Markdown. {{< key "#" >}} за которым следует слово. Вы увидите, что хэш в синтаксисе Markdown обозначает первый заголовок. Таким образом, если вы знаете, какой специальный символ вызывает то или иное форматирование, вы сможете захватывать и форматировать свои тексты гораздо быстрее.

В настоящее время можно вызвать только часть форматирования с помощью специального символа. Вы можете узнать, что это такое, в этой статье.

## Запуск форматирования синтаксиса в SeaTable с помощью специальных символов

В настоящее время в SeaTable можно активировать следующие опции синтаксиса с помощью специальных символов.

### Заголовки

Строка, начинающаяся с цифр от одного до трех {{< key "#" >}}\-символ, за которым следует символ пробела, приводит к результату **Заголовок** с первого по третий уровень.

**Синтаксис разметки**

\## Заголовок первого уровня  
\## Заголовок второго уровня  
\## Заголовок третьего уровня

**Результат**

![Результат заголовков синтаксиса Markdown](images/syntax-headings-richtext-markdown-1.png)

### Полужирный и курсив

Текст, написанный {{< key "\*" >}} или {{< key "\_" >}} Пара окружена **курсив** форматированный (что в Markdown означает подчеркивание); текст, который сопровождается двойным {{< key "\*\*" >}} или {{< key "\_ \_" >}} Пара окружена **полужирный** форматированный (что означает сильное подчеркивание в Markdown). Форматирование также можно комбинировать с другими встроенными форматами.

**Синтаксис разметки**

\*Этот текст будет набран курсивом\*.  
Это также будет курсив

\*\*Этот текст будет выделен жирным шрифтом\*\*.  
\_\_Это также будет выделено жирным шрифтом\_\_

\_\*\*Сочетайте их для получения полужирного курсива\*\*\_.

**Результат**

_Этот текст будет набран курсивом_  
_Это также будет курсив_

**Этот текст будет выделен жирным шрифтом**  
**Этот текст также будет жирным**

_**Объедините их для получения полужирного курсива**_

### Несортированный список

Вы можете либо {{< key "\-" >}} или {{< key "\*" >}} как маркер для **неупорядоченный список** использовать.

![Результат синтаксиса Markdown Несортированный список](images/result-unordered-list-syntax-1.png)

### Сортированный список

Если строка начинается с символа {{< key "1." >}}он автоматически преобразуется в **отсортированный список** преобразованный.

Пункт 1  
2\. пункт 2

### Котировки блоков

A **Блоквот** это строка, которая начинается с {{< key "\>" >}} а затем следует содержание цитаты. Цитаты отмечаются отступом и рамкой на левом поле.

![Уменьшенная цитата](images/markdown-cite.png)

### Инлайн-код

Текст, написанный {{< key "FIXME: backtick" >}} пара окружена, представляется как встроенный код. Таким образом, из **apiKey - это ваш секретный токен API** следующее форматирование:

![Результат инлайн-кода синтаксиса Markdown](images/result-inline-code.png)

### Кодовые блоки

A **Кодовый блок** достигается тремя последовательными **Backticks** {{< key "```" >}} введен. Особенностью блока кода является то, что содержание **Инлайн-код** отображается моноширинным шрифтом. Отступы также сохраняются точно в том виде, в котором они были введены.

![Блок кода в формате Markdown](images/markdown-codeblock.png)

### Разрывы строк

Каждый разрыв строки {{< key "↵" >}} рассматривается как жесткий перенос строки с интервалом между двумя строками. Разрыв строки без интервала между отдельными строками невозможен. Также **множественные переносы строк** удаляются и сводятся к простому переносу строки.

## Форматирование, которое нельзя активировать с помощью специальных символов

**Флажки** обычно представлены в Markdown с помощью {{< key "\[ \]" >}} или {{< key "\[x\]" >}} вводится. Однако этот ввод не преобразуется автоматически в флажок. Вместо этого необходимо нажать на маленький значок флажка {{< seatable-icon icon="dtable-icon-check-square" >}} нажать.

**Гиперссылки** также поддерживаются в SeaTable Markdown, но их можно вставлять только с помощью значка {{< seatable-icon icon="icon-link" >}}.

## Форматирование не поддерживается

{{< key "~~зачеркнуто~~" >}} не поддерживается SeaTable. Дублирующиеся переносы строк также удаляются, и в таблице появляется надпись {{< key "<br>" >}} не интерпретируется как дополнительный перевод строки.
